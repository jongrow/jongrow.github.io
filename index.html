Lambda概述.md

发表于 2016-06-28

What Lambda

维基百科对Lambda expression的解释： 不用绑定到一个标识符，并且可以被调用的函数。

a function (or a subroutine) defined, and possibly called,without being bound to an identifer.

简单点说：一段带有输入参数的可执行语句块。

简单示例看下使用lambda表达式和使用匿名内部类排序的不同效果

//使用lambda形式  省略list的构造
List<String> names = ...;   
Collections.sort(names, (o1, o2) -> o1.compareTo(o2));

//使用匿名内部类的形式 这里省略list的构造
List<String> names = ...;
Collections.sort(names, new Comparator<String>() {
      @Override
      public int compare(String o1, String o2) {
                return o1.compareTo(o2);
      }
});

Why suggest to use Lambda

Java8的lambda表达式给我提供了创建SAM(single abstract method)接口更近简单的语法。

firt of all ，我们先抽象一些Lambda表达式的一般语法

（Type1 param1,Type2 param2,...,TypeN paramN）->{
     statment1;
     statment2;
     ...
     return statmentM;    
 }

接下来我们看下Lambda表达式语法的各种简化版：

  1. 参数类型省略，绝大多数情况，编译器可以从上下文环境中推断中Lambda表达式的参数类型

    （param1,param2,...,paramN）-{
    statment1;
    statment2;
    ...
    return statmentM;
}

  2. 当lambda表达式的参数只有一个时，可以省略小括号。

    param1 -> {
    statment1;
    statment2;
    ...
    return statmentM;
}

  3. 当lambda表达式只包含一条语句时，可以省略大括号、return和语句结尾的分号

    param1 -> statment

  4. 使用Method Reference

Lambda表达式的三个重要组成部分

  • 输入参数
  • 可执行语句

  • 存放外部变量的空间

    Lambda表达式访问外部变量有一个非常重要的限制：变量不可变 （只是引用不可变，而不只是真正的不可变）

Lambda眼中的this

在Lamda中，this并不是指向lamdda表达式中产生的那个SAM对象，而是声明它的外部对象。

IM使用规范

使用指南

连接IM server

IMLogicManager.getInstance().connectIMServer();

断开与IM server的连接

IMLogicManager.getInstance().closeIMServer();

发送消息

IMLogicManager.getInstance.mMsgLogic.sendMessage(IMMessage msg);

收到消息

如需对接受到的消息进行处理，目前需要采用接受通知的形式来获取对应的消息,在对应的callback中进行处理。
NotifyManager.getInstance().registerNotifyCallback(NotifyKey.RECEIVE_CHAT_MSG,NotifyCallback callback);

结构设计

实际编码过程中，并不是完全按照该设计进行编码，会有一定的变化。

IM结构自上而下采用三层结构分为：展示层，控制层，数据层，分别对应内存对象，协议格式，二进制流处理，由于目前采用的是第三方IM快速接入。控制层和数据层的业务逻辑暂时不做处理，后续如有需求变更，可以迭代开发。

IMSocket 展示层（V）

对业务层提供统一的入口，所有对IM的访问统一要经过IMLogicManager如果是和消息Message相关的，会自动转投到IMMessageLogic，和用户相关的统一转投都IMUserLogic处理。

IMSocket 控制层（C）

进行与IM 服务器的长连接，保持通道的稳定性，及收发包队列的管理，上下文，对上层消息的回调及超时重连，重发机制的逻辑，暂未处理。

IMSocket 数据层 (M)

主要是收包线程与发包线程及主线程通信的相关处理及转换，暂未处理。

数据类型

内存对象格式

面向对象编程，业务开发者相对熟悉的数据格式

void sendMessage(IMMessage msg);
void sendMessage(IMMessage msg,Callback callback)

现在使用第三方RongCloud的快速接入，暂时上层业务只需要传入数据对象即可。

底层协议格式

上游业务传递的Java对象，在真正的IM控制层需要封装为实际开发采用的协议格式，方便统一处理。现在没有自建协议，留作TODO处理。

二进制流格式

IM通讯本质上通过与服务器采用TCP长连接，本质上无论采用何种协议，最终都需转化为二进制流在网络上传输，常见的做法是生成 定长包头+变长包体的二进制流协议，再对数据进行压缩，加密完成一系列处理后，通过socket投递给服务器。
对于返回的报文处理是一个逆序的过程。
